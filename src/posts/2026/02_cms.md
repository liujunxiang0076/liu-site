## 项目背景 

在尝试过多种开源 CMS 方案后，总觉得在**定制化灵活性**和**操作体验**上差了一口气。为了打造一个最契合自己工作流的后台管理系统，我决定从零开始，利用业余时间构建了这个项目。

经过近期的密集开发，目前系统架构已经基本跑通，UI 视觉也达到了预期效果。

## 技术栈选型

为了保证系统的现代感与高性能，我选择了目前主流的“黄金组合”：

* **前端框架**：Vue 3 + Vite 5 + TypeScript
* **UI 组件库**：Element Plus（配合 UnoCSS 进行样式微调）
* **状态管理**：Pinia
* **后端驱动**：Node.js + NestJS / Go (根据个人实际情况修改)
* **图标方案**：Iconify + Element Plus Icons

## 已实现的核心功能

1.  **权限控制 (RBAC)**：实现了完善的登录鉴权，支持动态路由生成，不同角色的菜单权限逻辑已闭环。
2.  **响应式布局**：侧边栏支持折叠，整体 UI 在不同分辨率下表现稳定，尤其是登录页的极简设计。
3.  **内容管理**：初步实现了文章列表展示、分类筛选以及基础的 CRUD 操作。
4.  **仪表盘统计**：主页集成了数据概览卡片，可以直观看到内容更新频率和统计维度。
5.  **图床集成**：配合自建图床，实现了流畅的图片上传与引用流程。

## 遇到的挑战与突破

在开发过程中，最耗时的是**动态菜单与面包屑的联动逻辑**。为了保证用户在层级较深的页面不迷路，我重写了基于路由表的递归算法。此外，在 UI 细节上，我反复调试了 Element Plus 的主题色，以确保整体视觉呈现出一种“高级灰”的科技感。

## 后续迭代计划

* [ ] 增加 Markdown 编辑器的高级扩展（如公式、图表支持）。
* [ ] 完善系统日志与操作审计功能。
* [ ] 接入多语言 (i18n) 模块。
* [ ] 进一步优化首屏加载速度。

**总结：**
虽然目前只是“小有所成”，但从 0 到 1 的过程让我对工程化架构有了更深的理解。代码还在不断打磨中，欢迎同道中人一起交流开发经验！
